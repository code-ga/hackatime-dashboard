## Notes

- The authenticated endpoints use `/authenticated/*` path prefix (not `/users/me` or similar)
- These endpoints require a Bearer token in the Authorization header (handled by `fetcher` if token is provided via init param, or manually in components)
- The existing `fetcher` utility doesn't include auth headers by default. The dashboard manually adds them. We may need to enhance `fetcher` to optionally accept auth token, or keep manual header addition in component code.
- Consider whether to add token handling to `fetcher` or keep it simple and let callers pass `init` options.

## Implementation Details (Updated)

### Auth Fetcher
Add to `lib/fetch.ts` or within `lib/hackatime.ts`:
```typescript
export function authFetcher<T>(input: RequestInfo, init?: RequestInit): Promise<T> {
  const token = localStorage.getItem("access_token");
  const headers = new Headers(init?.headers);
  if (token) {
    headers.set("Authorization", `Bearer ${token}`);
  }
  return fetch(input, { ...init, headers }).then((res) => {
    if (!res.ok) {
      throw new Error(`HTTP error! status: ${res.status}`);
    }
    return res.json() as Promise<T>;
  });
}
```
