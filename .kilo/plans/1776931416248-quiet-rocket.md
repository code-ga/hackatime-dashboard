# Plan to Fix Crosshair Bug in Pet PiP Feature

## Problem Description
The crosshair (target reticle) should be fixed at the center of the PiP window but currently doesn't move with the window. It's randomly positioned on screen and stays static, causing misalignment when the PiP window moves.

## Root Cause
- The target position is set randomly via `moveTargetToRandom()` and remains fixed on screen coordinates.
- When the PiP window moves, the target doesn't update to stay centered in the new window position.
- No mechanism to keep target centered relative to the PiP window.

## Solution Overview
Make the target reticle fixed at the center of the PiP window:
1. Update target position whenever PiP window moves to maintain center alignment.
2. Modify initialization to set target at PiP center initially.
3. Remove or adjust random positioning since target should be centered.

## Implementation Steps

### 1. Update Target Positioning Logic
- File: `components/PetPiP.tsx`
- In `trackMovement` callback, after updating pipWindow position, recalculate target to center of window.
- Set target.x = pipWindow.x + PIP_WIDTH / 2
- Set target.y = pipWindow.y + PIP_HEIGHT / 2

### 2. Modify Target Initialization
- File: `components/PetPiP.tsx`
- In `handleOpenPet`, after starting animation, set initial target position to center of PiP window.
- Remove or modify random target placement.

### 3. Update Pet World State Management
- File: `lib/pet-world.ts`
- Add `updateTargetPosition(x: number, y: number)` function for programmatic updates.
- Optionally modify `moveTargetToRandom()` to center on PiP if needed, or remove random logic.

### 4. Testing
- Verify target stays at center of PiP window when window moves.
- Confirm scoring works when pet enters centered target zone.
- Test window resizing maintains center alignment.

## Expected Behavior After Fix
- Crosshair remains fixed at the center of the PiP window regardless of window movement.
- Scoring triggers when pet is visible and within the centered target radius.
- Pet teleportation works as before, target stays centered for next round.

## Files to Modify

- `lib/pet-world.ts`: Add updateTargetPosition function
