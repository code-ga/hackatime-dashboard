# Screen Portal Effect - Plan

## Overview

Implement a "Screen Portal" effect where a virtual pet exists in global screen coordinates and is visible through a draggable Picture-in-Picture window acting as a viewing portal.

## Architecture Decision: SharedState Pattern

Since the PiP window is same-origin with the opener, we can share a JavaScript object directly (no SharedWorker/BroadcastChannel needed). The main document's animation loop drives all state; the PiP window's canvas simply reads from shared state and renders.

## Core Components

### 1. Shared Pet State (`lib/pet-world.ts`)

```typescript
interface PetWorldState {
  // Pet position in global screen coordinates
  pet: { x: number; y: number; velocityX: number; velocityY: number };
  // PiP window position
  pipWindow: { x: number; y: number; width: number; height: number };
  // Velocity tracking for inertia
  pipVelocity: { x: number; y: number };
}
```

### 2. Pet Wander System

The pet wanders randomly across the entire screen using:
- Random target selection within screen bounds
- Smooth interpolation toward target
- Boundary clamping to screen edges

### 3. Velocity Detection

Track PiP window movement deltas over time to calculate velocity:
- Store last position and timestamp
- On each update, compute `(newPos - oldPos) / deltaTime`
- Apply exponential smoothing for stability

### 4. Portal Logic (PiP Canvas Rendering)

Calculate pet's position relative to PiP window:
```typescript
localX = pet.x - pipWindow.x
localY = pet.y - pipWindow.y
```

Visibility check:
- If `localX < 0 || localX > width || localY < 0 || localY > height` → Pet is outside window

Direction Arrow:
- Draw arrow pointing toward pet's actual global position
- Arrow positioned at window edge, pointing outward

## Implementation Steps

### Step 1: Create `lib/pet-world.ts`

Shared state module with:
- `PetWorldState` interface
- Singleton state object
- Exports for reading/updating state

### Step 2: Create `lib/pet-animation.ts`

Main animation loop module with:

1. **Wander System**
   - Random target generation within screen bounds
   - Smooth lerp movement toward target
   - Velocity damping

2. **PiP Velocity Tracker**
   - Compare PiP window position to last known position
   - Compute velocity with time delta
   - Exponential moving average for smoothing

3. **Inertia Physics**
   - Apply negative PiP velocity to pet
   - `pet.velocityX -= pipVelocity.x * INERTIA_FACTOR`
   - `pet.velocityY -= pipVelocity.y * INERTIA_FACTOR`
   - This creates the "sliding left when window jerks right" effect

4. **Position Update Loop**
   - `requestAnimationFrame` loop
   - Update all positions each frame
   - Sync to shared state

### Step 3: Update `components/PetPiP.tsx`

1. Replace React-based rendering with Canvas-based PiP content
2. Move all animation logic to `lib/pet-animation.ts`
3. Pass shared state to PiP content via closure (same JavaScript realm)

### Step 4: Create `components/PetCanvas.tsx`

Canvas component for PiP window that:
1. Renders pet SVG at calculated local position
2. Draws direction arrow when pet is outside
3. Handles canvas resize

### Step 5: Update `components/PetSvg.tsx`

Optional: Convert to pure SVG generation for canvas drawing, or create a separate canvas drawing function.

## Physics Constants

```typescript
const INERTIA_FACTOR = 0.3;        // How much window velocity affects pet
const PET_SPEED = 2;              // Pet wander speed
const VELOCITY_DECAY = 0.95;       // Velocity damping per frame
const WANDER_INTERVAL = 3000;     // New target every 3 seconds
const ARROW_SIZE = 20;            // Direction arrow dimensions
```

## Key Formulas

### Pet Position Update
```
pet.x += pet.velocityX
pet.y += pet.velocityY
pet.velocityX *= VELOCITY_DECAY
pet.velocityY *= VELOCITY_DECAY
```

### Inertia (Negative Force)
```
pet.velocityX -= pipVelocity.x * INERTIA_FACTOR
pet.velocityY -= pipVelocity.y * INERTIA_FACTOR
```

### Portal Visibility Check
```
isVisible = localX >= 0 && localX <= windowWidth && localY >= 0 && localY <= windowHeight
```

### Direction Arrow Angle
```
angle = atan2(pet.y - window.y, pet.x - window.x)
```
