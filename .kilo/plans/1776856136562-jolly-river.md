# Dashboard Analytics Feature Implementation Plan

## Overview
Implement three main features for the HackaTime dashboard: Date/Filter system, Analytics Charts, and Timeline visualization.

---

## API Capabilities Analysis

### Available Endpoints

| Endpoint | Filters Supported |
|---------|-----------------|
| `/users/{username}/stats` | `start_date`, `end_date`, `filter_by_project`, `filter_by_category`, `features` |
| `/users/{username}/heartbeats/spans` | `start_date`, `end_date`, `project`, `filter_by_project` |
| `/authenticated/projects` | `start_date`, `end_date`, `since`, `until`, `projects` |
| `/authenticated/hours` | `start_date`, `end_date` |

### Missing API Support
- **OS filter**: Not directly supported in API query params
- **Editor filter**: Not directly supported in API query params
- **OS chart**: Not in stats response (need to use `heartbeats/latest` or create proxy)
- **Editor chart**: Available via `features=editors`

---

## Feature 1: Filters (Date Range, Project, Language, OS, Editor, Category)

### Components to Create
- `components/dashboard/filter-bar.tsx` - Filter controls

### Filter Options
| Filter | Source | Implementation |
|--------|--------|---------------|
| Date Range | `start_date`, `end_date` | Built-in API support |
| Project | `filter_by_project` | Built-in API support + dropdown from projects list |
| Language | `filter_by_category` | Partial (uses category like "ai+coding", "coding") |
| Category | `filter_by_category` | Built-in API support |
| OS | N/A | Display only from current heartbeat |
| Editor | N/A | Display only from current heartbeat |

### Implementation
- Add date range picker (calendar inputs for start_date, end_date)
- Add project dropdown (fetch from `/authenticated/projects`)
- Add language/category dropdown
- Filter state managed in dashboard page (URL params or local state)

## Feature 2: Analytics Charts (Language, Editor, OS Distribution)

### Components to Create
- `components/dashboard/language-chart.tsx` - Doughnut/bar chart for languages
- `components/dashboard/editor-chart.tsx` - Doughnut/bar chart for editors  
- `components/dashboard/os-chart.tsx` - Doughnut/bar chart for OS

### Implementation
- **Language Chart**: Use `stats.languages[]` - already in API response with `features=languages`
- **Editor Chart**: Use `stats.editors[]` - requires `features=editors` query param
- **OS Chart**: NOT available in stats. Alternative: Use `/authenticated/hours` data + mock, OR create separate API call
  - Option: Track current user's OS from latest heartbeat as proxy
  - Better: This is a limitation - OS data not readily available

### Chart Library
- Use existing Recharts or create simple SVG pie/donut charts (existing codebase pattern)

## Feature 3: Timeline (Project Timeline & Activity Timeline)

### Components to Modify
- `components/user/activity-timeline.tsx` - Already exists, need to enhance
- Add project-specific timeline view

### Implementation
- **Activity Timeline**: Use `getAnyUserHeartbeatsSpans` - provides spans with `start_time`, `end_time`, `duration`
- **Project Timeline**: Filter spans by project using `filter_by_project` param
- Visualize as timeline bars showing coding sessions by day/week

### Timeline Features
- Date-grouped activity blocks
- Hover for duration/project details
- Color-coded by project or category

## Limitations & Notes

1. **OS/Editor Filter**: Not supported in API - will show display only
2. **OS Chart**: Not available in stats API - may need proxy or mock data
3. **Language Filter**: Uses `filter_by_category` which maps to categories like "ai+coding" not individual languages
4. Date range filters work on all endpoints
