# Dark Mode Implementation Plan

## Goal
Add dark mode toggle button to the dashboard application.

## Analysis
- Project uses Tailwind CSS v4 with CSS variables for theming
- Dark mode CSS variables already defined in `app/globals.css` (lines 89-124)
- Currently no theme provider or toggle mechanism exists
- Project uses `@phosphor-icons/react` for icons (available: `Moon`, `Sun`)

## Implementation Steps

### 1. Install next-themes package
```
bun add next-themes
```

### 2. Create ThemeProvider component
- File: `components/theme-provider.tsx`
- Wraps app with `next-themes` Provider
- Enables system preference detection
- Stores preference in localStorage

### 3. Update RootLayout
- File: `app/layout.tsx`
- Import and wrap children with ThemeProvider
- Add `suppressHydrationWarning` to html element (required by next-themes)

### 4. Create ThemeToggle button component
- File: `components/theme-toggle.tsx`
- Uses Phosphor icons (Sun/Moon)
- Button variant: ghost or outline
- Position: Can be added to dashboard header or as floating button

### 5. Add ThemeToggle to pages
- Add to `app/dashboard/page.tsx`
- Add to `app/user/[username]/page.tsx`

## Files to Modify
1. `package.json` - add next-themes dependency
2. `components/theme-provider.tsx` - create new
3. `components/theme-toggle.tsx` - create new
4. `app/layout.tsx` - wrap with ThemeProvider
5. `app/dashboard/page.tsx` - add toggle
6. `app/user/[username]/page.tsx` - add toggle

## Notes
- Dark mode colors already defined in globals.css
