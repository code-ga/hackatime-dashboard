# Plan: Integrate Authenticated API Data into Dashboard UI

## Goal
Replace the current static dashboard UI with real data from the authenticated Hackatime API endpoints.

## Available Authenticated Endpoints (from lib/hackatime.ts)

| Endpoint | Returns |
|----------|---------|
| `hackatimeApi.authenticated.getMe()` | User profile (github_username, emails, trust_factor) |
| `hackatimeApi.authenticated.getHours()` | daily_average, monthly_average, total |
| `hackatimeApi.authenticated.getStreak()` | current_streak, longest_streak |
| `hackatimeApi.authenticated.getProjects()` | projects array with name, total_seconds, percent |
| `hackatimeApi.authenticated.getLatestHeartbeat()` | Current activity info |
| `hackatimeApi.authenticated.getApiKeys()` | API token |

## Implementation Plan

### 1. Update imports in page.tsx
- Import `hackatimeApi` from `@/lib/hackatime`
- Import types from `@/types/hackatime`

### 2. Replace useEffect data fetching
- Remove direct fetch call to `/authenticated/me`
- Use `hackatimeApi.authenticated.getMe()` instead
- Add calls to other endpoints: `getHours()`, `getStreak()`, `getProjects()`, `getLatestHeartbeat()`

### 3. Create state for all authenticated data
```typescript
const [user, setUser] = useState<GetAuthenticatedMeResponse | null>(null);
const [hours, setHours] = useState<GetAuthenticatedHoursResponse | null>(null);
const [streak, setStreak] = useState<GetAuthenticatedStreakResponse | null>(null);
const [projects, setProjects] = useState<GetAuthenticatedProjectsResponse | null>(null);
const [latestHeartbeat, setLatestHeartbeat] = useState<GetAuthenticatedLatestHeartbeatResponse | null>(null);
```

### 4. Update UI to display all data
- Keep existing user card with github_username, email, trust_level
- Add Hours card showing daily/monthly average and total
- Add Streak card showing current and longest streak
- Add Projects list (reuse existing ProjectList component pattern)
- Add Latest Activity card showing current coding activity

### 5. Error handling
- Handle case when token is missing or expired
- Show appropriate loading states

## Note
- The `authFetcher` reads token from localStorage automatically
- Only works in client components (already using "use client")
